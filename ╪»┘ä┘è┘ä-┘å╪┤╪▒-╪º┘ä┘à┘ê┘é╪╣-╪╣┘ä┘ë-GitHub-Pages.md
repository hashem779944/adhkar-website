# دليل شامل لنشر موقع أذكار الصباح والمساء على GitHub Pages

**التاريخ:** 7 أغسطس 2025

## مقدمة

يعد GitHub Pages واحداً من أفضل الخدمات المجانية لاستضافة المواقع الثابتة، حيث يوفر استضافة موثوقة وسريعة مع إمكانية ربط نطاق مخصص. في هذا الدليل الشامل، ستتعلم كيفية نشر موقع أذكار الصباح والمساء الذي تم تطويره خصيصاً لك على GitHub Pages خطوة بخطوة.

GitHub Pages هو خدمة استضافة مجانية تقدمها شركة GitHub تتيح لك نشر المواقع الثابتة مباشرة من مستودعات GitHub الخاصة بك. هذه الخدمة مثالية للمشاريع الشخصية والمواقع التعليمية والمشاريع مفتوحة المصدر، وهي تدعم HTML وCSS وJavaScript بشكل كامل، مما يجعلها الخيار الأمثل لموقع الأذكار الذي طورناه.

الموقع الذي ستقوم بنشره يتميز بتصميم عصري وبسيط، مع ثيمين مختلفين يناسبان أوقات الصباح والمساء. التصميم متجاوب ويعمل بشكل مثالي على جميع الأجهزة، من الهواتف الذكية إلى أجهزة الكمبيوتر المكتبية. كما أن الموقع يحتوي على جميع الأذكار الأساسية من حصن المسلم مع التشكيل الكامل، مما يجعله مرجعاً موثوقاً للمسلمين في جميع أنحاء العالم.




## المتطلبات الأساسية

قبل البدء في عملية النشر، تحتاج إلى التأكد من توفر المتطلبات التالية على جهازك وحسابك:

### حساب GitHub

أول ما تحتاجه هو حساب مجاني على GitHub. إذا لم يكن لديك حساب بعد، يمكنك إنشاء واحد بسهولة من خلال زيارة موقع GitHub الرسمي على الرابط [github.com](https://github.com). عملية التسجيل بسيطة وتتطلب فقط عنوان بريد إلكتروني صالح واختيار اسم مستخدم فريد وكلمة مرور قوية.

عند إنشاء الحساب، ستحتاج إلى التحقق من عنوان البريد الإلكتروني الخاص بك. هذه خطوة مهمة لأنها تضمن أمان حسابك وتمكنك من الوصول إلى جميع ميزات GitHub، بما في ذلك GitHub Pages. بعد التحقق من البريد الإلكتروني، ستتمكن من الوصول إلى لوحة التحكم الرئيسية في GitHub.

### متصفح ويب حديث

تحتاج إلى متصفح ويب حديث يدعم جميع ميزات GitHub الحديثة. ننصح باستخدام أحدث إصدارات من Google Chrome أو Mozilla Firefox أو Microsoft Edge أو Safari. هذه المتصفحات توفر أفضل تجربة استخدام مع واجهة GitHub وتضمن عمل جميع الوظائف بشكل صحيح.

### ملفات الموقع

ستحتاج إلى جميع ملفات الموقع التي تم تطويرها، والتي تشمل:

- `index.html` - الصفحة الرئيسية التي تحتوي على أزرار الانتقال بين صفحتي الصباح والمساء
- `morning.html` - صفحة أذكار الصباح بالثيم النهاري الفاتح
- `evening.html` - صفحة أذكار المساء بالثيم الليلي المظلم
- `morning-style.css` - ملف التنسيق الخاص بصفحة الصباح
- `evening-style.css` - ملف التنسيق الخاص بصفحة المساء
- `morning-script.js` - ملف JavaScript لوظائف صفحة الصباح
- `evening-script.js` - ملف JavaScript لوظائف صفحة المساء
- `README.md` - ملف التوثيق الذي يشرح المشروع

هذه الملفات تشكل موقعاً متكاملاً يعمل بشكل مستقل دون الحاجة إلى خادم خلفي أو قاعدة بيانات، مما يجعله مثالياً للاستضافة على GitHub Pages.

### معرفة أساسية بـ Git (اختيارية)

رغم أن معرفة Git ليست ضرورية للنشر الأساسي على GitHub Pages، إلا أن فهم المفاهيم الأساسية لـ Git سيساعدك في إدارة موقعك بشكل أفضل مستقبلاً. Git هو نظام إدارة الإصدارات الذي يستخدمه GitHub، وهو يتيح لك تتبع التغييرات في ملفاتك والعودة إلى إصدارات سابقة عند الحاجة.

إذا كنت مبتدئاً في Git، لا تقلق. سنقوم بشرح جميع الخطوات بالتفصيل، وستتمكن من نشر موقعك حتى لو لم تكن تعرف Git من قبل. GitHub يوفر واجهة ويب سهلة الاستخدام تتيح لك القيام بمعظم العمليات دون الحاجة إلى استخدام سطر الأوامر.


## الخطوة الأولى: إنشاء مستودع جديد على GitHub

### تسجيل الدخول إلى GitHub

ابدأ بتسجيل الدخول إلى حسابك على GitHub من خلال زيارة [github.com](https://github.com) وإدخال بيانات الدخول الخاصة بك. بعد تسجيل الدخول بنجاح، ستجد نفسك في الصفحة الرئيسية لحسابك، والتي تعرض نظرة عامة على نشاطك ومستودعاتك الحالية.

### إنشاء مستودع جديد

لإنشاء مستودع جديد، ابحث عن زر "New" الأخضر في الجانب الأيسر من الصفحة الرئيسية، أو يمكنك النقر على علامة "+" في الزاوية العلوية اليمنى من الصفحة واختيار "New repository" من القائمة المنسدلة. هذا سيأخذك إلى صفحة إنشاء مستودع جديد.

### تكوين إعدادات المستودع

في صفحة إنشاء المستودع الجديد، ستحتاج إلى ملء عدة حقول مهمة:

**اسم المستودع (Repository name):** اختر اسماً وصفياً لمستودعك. ننصح باستخدام اسم مثل "adhkar-website" أو "morning-evening-adhkar" أو أي اسم آخر يصف محتوى الموقع بوضوح. تجنب استخدام المسافات في الاسم، واستخدم الشرطات (-) أو الشرطات السفلية (_) بدلاً من ذلك.

**الوصف (Description):** أضف وصفاً مختصراً للمستودع، مثل "موقع أذكار الصباح والمساء من حصن المسلم" أو "Website for morning and evening Islamic supplications". هذا الوصف سيساعد الآخرين على فهم محتوى مستودعك.

**الخصوصية (Visibility):** اختر "Public" لجعل المستودع عاماً ومتاحاً للجميع. هذا ضروري لاستخدام GitHub Pages المجاني. إذا اخترت "Private"، ستحتاج إلى اشتراك مدفوع لاستخدام GitHub Pages.

**تهيئة المستودع:** تأكد من تحديد خيار "Add a README file" لإنشاء ملف README تلقائياً. يمكنك أيضاً اختيار إضافة ملف .gitignore إذا كنت تريد استبعاد ملفات معينة من المستودع، لكن هذا ليس ضرورياً لمشروعنا الحالي.

بعد ملء جميع الحقول المطلوبة، انقر على زر "Create repository" الأخضر في أسفل الصفحة. سيقوم GitHub بإنشاء المستودع الجديد وتوجيهك إلى صفحة المستودع الرئيسية.

### فهم واجهة المستودع

بعد إنشاء المستودع، ستجد نفسك في الصفحة الرئيسية للمستودع. هذه الصفحة تحتوي على عدة عناصر مهمة:

- **شريط التنقل العلوي:** يحتوي على اسم المستودع ومعلومات أساسية مثل عدد النجوم والمتابعين
- **علامات التبويب:** تشمل "Code" للملفات، "Issues" للمشاكل، "Pull requests" للطلبات، "Actions" للأتمتة، "Projects" للمشاريع، "Wiki" للتوثيق، "Security" للأمان، "Insights" للإحصائيات، و"Settings" للإعدادات
- **منطقة الملفات:** تعرض جميع الملفات والمجلدات في المستودع
- **منطقة README:** تعرض محتوى ملف README.md إذا كان موجوداً

هذه الواجهة ستكون مركز إدارة موقعك، حيث ستقوم برفع الملفات وتحديثها وتكوين إعدادات GitHub Pages.


## الخطوة الثانية: رفع ملفات الموقع إلى المستودع

### طرق رفع الملفات

يوفر GitHub عدة طرق لرفع الملفات إلى المستودع. سنركز على الطريقة الأسهل والأكثر ملاءمة للمبتدئين، وهي استخدام واجهة الويب الخاصة بـ GitHub.

### رفع الملفات عبر واجهة الويب

#### الطريقة الأولى: رفع الملفات واحداً تلو الآخر

في الصفحة الرئيسية لمستودعك، ستجد زر "Add file" في الجانب الأيمن من منطقة الملفات. انقر على هذا الزر واختر "Upload files" من القائمة المنسدلة. سيأخذك هذا إلى صفحة رفع الملفات.

في صفحة رفع الملفات، يمكنك سحب وإفلات الملفات مباشرة من مجلد الموقع على جهازك إلى المنطقة المخصصة، أو يمكنك النقر على "choose your files" لتحديد الملفات يدوياً من خلال متصفح الملفات.

ابدأ برفع الملفات التالية بالترتيب:

1. **index.html** - الصفحة الرئيسية
2. **morning.html** - صفحة أذكار الصباح  
3. **evening.html** - صفحة أذكار المساء
4. **morning-style.css** - تنسيق الثيم الصباحي
5. **evening-style.css** - تنسيق الثيم المسائي
6. **morning-script.js** - وظائف صفحة الصباح
7. **evening-script.js** - وظائف صفحة المساء

بعد تحديد الملفات، ستظهر قائمة بالملفات المحددة للرفع. في أسفل الصفحة، ستجد قسم "Commit changes" حيث يمكنك إضافة رسالة توضح ما تقوم برفعه. اكتب رسالة وصفية مثل "إضافة ملفات موقع أذكار الصباح والمساء" في الحقل الأول.

يمكنك أيضاً إضافة وصف أكثر تفصيلاً في الحقل الثاني إذا كنت تريد توثيق التغييرات بشكل أكبر. بعد كتابة الرسالة، انقر على زر "Commit changes" الأخضر لحفظ الملفات في المستودع.

#### الطريقة الثانية: رفع جميع الملفات دفعة واحدة

إذا كنت تفضل رفع جميع الملفات في عملية واحدة، يمكنك تحديد جميع ملفات الموقع مرة واحدة وسحبها إلى منطقة الرفع. هذه الطريقة أسرع وأكثر كفاءة، خاصة إذا كان لديك عدد كبير من الملفات.

عند رفع عدة ملفات في نفس الوقت، تأكد من أن جميع الملفات في نفس المجلد وأنها لا تحتوي على مجلدات فرعية معقدة. موقع الأذكار الذي طورناه بسيط ولا يحتوي على مجلدات فرعية، لذا يمكنك رفع جميع الملفات بأمان.

### التحقق من رفع الملفات بنجاح

بعد إتمام عملية الرفع، ستعود إلى الصفحة الرئيسية للمستودع حيث ستجد جميع الملفات مدرجة في قائمة الملفات. تأكد من وجود جميع الملفات المطلوبة:

- ✅ index.html
- ✅ morning.html  
- ✅ evening.html
- ✅ morning-style.css
- ✅ evening-style.css
- ✅ morning-script.js
- ✅ evening-script.js
- ✅ README.md

يمكنك النقر على أي ملف لعرض محتواه والتأكد من أنه تم رفعه بشكل صحيح. إذا وجدت أي مشكلة في أحد الملفات، يمكنك النقر على أيقونة القلم (Edit) لتحرير الملف مباشرة من واجهة GitHub.

### إدارة الملفات والتحديثات المستقبلية

بعد رفع الملفات الأولية، قد تحتاج إلى تحديث محتوى الموقع أو إضافة ملفات جديدة في المستقبل. GitHub يجعل هذه العملية سهلة من خلال نفس واجهة الويب.

لتحديث ملف موجود، انقر على اسم الملف في قائمة الملفات، ثم انقر على أيقونة القلم في الزاوية العلوية اليمنى من عارض الملف. سيفتح هذا محرر النصوص المدمج في GitHub حيث يمكنك إجراء التعديلات المطلوبة.

لإضافة ملفات جديدة، استخدم نفس عملية "Add file" > "Upload files" التي استخدمتها في البداية. يمكنك أيضاً إنشاء ملفات جديدة مباشرة في GitHub باستخدام خيار "Create new file".

تذكر أن كل تغيير تقوم به في GitHub يُسمى "commit"، وكل commit يجب أن يحتوي على رسالة توضح ما تم تغييره. هذا يساعد في تتبع تاريخ التغييرات ويجعل إدارة المشروع أسهل على المدى الطويل.


## الخطوة الثالثة: تفعيل GitHub Pages

### الوصول إلى إعدادات المستودع

بعد رفع جميع ملفات الموقع بنجاح، حان الوقت لتفعيل GitHub Pages لجعل موقعك متاحاً على الإنترنت. ابدأ بالانتقال إلى علامة تبويب "Settings" في الجزء العلوي من صفحة المستودع. هذه العلامة موجودة في أقصى اليمين من شريط علامات التبويب.

إذا لم تجد علامة تبويب "Settings"، تأكد من أنك مالك المستودع أو لديك صلاحيات إدارية عليه. فقط مالكو المستودعات والمتعاونون الذين لديهم صلاحيات إدارية يمكنهم الوصول إلى إعدادات المستودع.

### العثور على إعدادات GitHub Pages

في صفحة الإعدادات، ستجد قائمة طويلة من الخيارات في الشريط الجانبي الأيسر. ابحث عن قسم "Code and automation" وانقر على "Pages" تحته. سيأخذك هذا إلى صفحة إعدادات GitHub Pages الخاصة بمستودعك.

إذا كانت هذه هي المرة الأولى التي تصل فيها إلى هذه الصفحة، ستجد رسالة تفيد بأن GitHub Pages غير مفعل حالياً لهذا المستودع. هذا أمر طبيعي ومتوقع للمستودعات الجديدة.

### تكوين مصدر GitHub Pages

في قسم "Source"، ستجد قائمة منسدلة تحتوي على عدة خيارات. الخيار الافتراضي عادة ما يكون "None" مما يعني أن GitHub Pages غير مفعل. انقر على القائمة المنسدلة واختر "Deploy from a branch".

بعد اختيار "Deploy from a branch"، ستظهر خيارات إضافية لتحديد الفرع (Branch) والمجلد الذي تريد نشر الموقع منه. في معظم الحالات، ستريد اختيار:

- **Branch:** main (أو master إذا كان هذا هو الاسم الافتراضي لفرعك الرئيسي)
- **Folder:** / (root) - هذا يعني أن الموقع سيتم نشره من الجذر الرئيسي للمستودع

إذا كانت ملفات موقعك في مجلد فرعي، يمكنك اختيار "/docs" بدلاً من ذلك، لكن في حالتنا، الملفات موجودة في الجذر الرئيسي للمستودع.

### حفظ الإعدادات

بعد تحديد الخيارات المناسبة، انقر على زر "Save" لحفظ الإعدادات. سيقوم GitHub بمعالجة طلبك وبدء عملية نشر الموقع. هذه العملية قد تستغرق بضع دقائق، خاصة في المرة الأولى.

بعد حفظ الإعدادات، ستظهر رسالة في أعلى الصفحة تفيد بأن موقعك قيد النشر. ستتضمن هذه الرسالة أيضاً الرابط الذي سيكون متاحاً لموقعك بمجرد اكتمال عملية النشر.

### فهم عملية النشر

عندما تقوم بتفعيل GitHub Pages، يحدث عدة أشياء خلف الكواليس:

1. **إنشاء فرع gh-pages:** GitHub قد ينشئ فرعاً خاصاً يسمى "gh-pages" يحتوي على الملفات المنشورة
2. **معالجة الملفات:** GitHub يتحقق من جميع ملفاتك ويتأكد من أنها صالحة للنشر
3. **إنشاء الموقع:** يتم إنشاء نسخة من موقعك على خوادم GitHub
4. **تخصيص الرابط:** يتم تخصيص رابط فريد لموقعك بالصيغة: `https://username.github.io/repository-name`

### التحقق من حالة النشر

يمكنك متابعة تقدم عملية النشر من خلال علامة تبويب "Actions" في مستودعك. هذه العلامة تعرض جميع العمليات التلقائية التي يقوم بها GitHub، بما في ذلك نشر موقعك على GitHub Pages.

في صفحة Actions، ستجد قائمة بجميع "workflows" أو سير العمل. ابحث عن workflow يسمى "pages build and deployment" أو شيء مشابه. انقر عليه لرؤية تفاصيل عملية النشر.

إذا كانت العملية ناجحة، ستجد علامة خضراء بجانب اسم العملية. إذا كانت هناك مشكلة، ستجد علامة حمراء مع تفاصيل الخطأ التي يمكن أن تساعدك في حل المشكلة.

### الحصول على رابط الموقع

بمجرد اكتمال عملية النشر بنجاح، ارجع إلى صفحة إعدادات GitHub Pages. ستجد الآن رسالة تفيد بأن موقعك منشور ومتاح على رابط معين. هذا الرابط عادة ما يكون بالصيغة:

`https://yourusername.github.io/your-repository-name`

على سبيل المثال، إذا كان اسم المستخدم الخاص بك "ahmad123" واسم المستودع "adhkar-website"، فسيكون رابط موقعك:

`https://ahmad123.github.io/adhkar-website`

انقر على هذا الرابط للتأكد من أن موقعك يعمل بشكل صحيح. يجب أن تظهر الصفحة الرئيسية لموقع الأذكار مع الأزرار للانتقال بين صفحتي الصباح والمساء.


## الخطوة الرابعة: اختبار الموقع وحل المشاكل الشائعة

### اختبار الوظائف الأساسية

بعد نشر موقعك بنجاح، من المهم إجراء اختبار شامل للتأكد من أن جميع الوظائف تعمل كما هو متوقع. ابدأ بزيارة الرابط الرئيسي لموقعك والتحقق من النقاط التالية:

**اختبار الصفحة الرئيسية:**
- تأكد من ظهور العنوان الرئيسي "أذكار الصباح والمساء" بوضوح
- تحقق من وجود زرين للانتقال إلى صفحتي الصباح والمساء
- تأكد من أن الألوان والتدرجات تظهر بشكل صحيح
- اختبر استجابة الصفحة على أحجام شاشات مختلفة

**اختبار صفحة أذكار الصباح:**
انقر على زر "أذكار الصباح" وتحقق من:
- ظهور الثيم النهاري بالألوان الذهبية والبرتقالية
- عرض جميع الأذكار مع التشكيل الصحيح
- وجود أزرار التنقل في أعلى الصفحة
- عمل التأثيرات المتحركة عند تحميل الصفحة
- ظهور عدد مرات قراءة كل ذكر بوضوح

**اختبار صفحة أذكار المساء:**
انقر على زر "أذكار المساء" وتحقق من:
- ظهور الثيم الليلي بالألوان الزرقاء والبنفسجية
- عرض تأثير النجوم في الخلفية
- صحة محتوى الأذكار المسائية
- عمل أزرار التنقل بشكل صحيح

### اختبار التوافق مع الأجهزة المختلفة

موقع الأذكار مصمم ليكون متجاوباً مع جميع أحجام الشاشات. اختبر موقعك على:

**أجهزة الكمبيوتر المكتبية:**
- افتح الموقع في متصفحات مختلفة (Chrome, Firefox, Safari, Edge)
- جرب تغيير حجم نافذة المتصفح للتأكد من استجابة التصميم
- تحقق من وضوح النصوص وسهولة القراءة

**الأجهزة اللوحية:**
- اختبر الموقع على جهاز iPad أو جهاز Android لوحي
- تأكد من أن الأزرار كبيرة بما يكفي للمس
- تحقق من أن التخطيط يتكيف بشكل مناسب مع الشاشة الأفقية والعمودية

**الهواتف الذكية:**
- اختبر الموقع على هواتف iPhone و Android
- تأكد من أن النصوص قابلة للقراءة دون الحاجة للتكبير
- تحقق من سهولة التنقل بين الصفحات باللمس

### حل المشاكل الشائعة

#### مشكلة عدم ظهور الموقع (404 Error)

إذا واجهت خطأ 404 عند زيارة رابط موقعك، جرب الحلول التالية:

1. **تحقق من اسم الملف الرئيسي:** تأكد من أن الملف الرئيسي يسمى "index.html" بالضبط (بأحرف صغيرة)
2. **انتظر وقتاً أطول:** أحياناً تستغرق عملية النشر وقتاً أطول من المتوقع، انتظر 10-15 دقيقة إضافية
3. **تحقق من إعدادات GitHub Pages:** ارجع إلى إعدادات Pages وتأكد من أن المصدر محدد بشكل صحيح
4. **امسح ذاكرة التخزين المؤقت:** امسح cache المتصفح أو جرب فتح الموقع في نافذة خاصة

#### مشكلة عدم تحميل ملفات CSS أو JavaScript

إذا كان الموقع يظهر لكن بدون تنسيق أو وظائف تفاعلية:

1. **تحقق من مسارات الملفات:** تأكد من أن جميع ملفات CSS و JavaScript في نفس مجلد index.html
2. **تحقق من أسماء الملفات:** تأكد من أن أسماء الملفات في كود HTML تطابق أسماء الملفات المرفوعة تماماً
3. **تحقق من حساسية الأحرف:** GitHub Pages حساس لحالة الأحرف، تأكد من التطابق الدقيق

#### مشكلة عدم ظهور الخطوط العربية بشكل صحيح

إذا كانت النصوص العربية تظهر بشكل غير صحيح:

1. **تحقق من ترميز الملفات:** تأكد من أن جميع ملفات HTML محفوظة بترميز UTF-8
2. **تحقق من اتجاه النص:** تأكد من وجود `dir="rtl"` في عنصر HTML
3. **تحقق من تحميل خط Amiri:** تأكد من أن رابط Google Fonts يعمل بشكل صحيح

### تحسين الأداء

لضمان أفضل أداء لموقعك:

**تحسين الصور:**
إذا أضفت صوراً لاحقاً، تأكد من ضغطها لتقليل حجم الملف وتحسين سرعة التحميل.

**تقليل حجم ملفات CSS:**
يمكنك استخدام أدوات ضغط CSS لتقليل حجم ملفات التنسيق، مما يحسن سرعة تحميل الموقع.

**اختبار سرعة الموقع:**
استخدم أدوات مثل Google PageSpeed Insights لاختبار سرعة موقعك والحصول على اقتراحات للتحسين.

### مراقبة الموقع

بعد نشر موقعك، من المهم مراقبته بانتظام:

**تحقق من الإحصائيات:**
يمكنك استخدام Google Analytics أو أدوات أخرى لمراقبة زوار موقعك وفهم كيفية استخدامهم له.

**تحديثات دورية:**
راجع محتوى الأذكار بانتظام وتأكد من صحته، وقم بتحديث التصميم حسب الحاجة.

**النسخ الاحتياطية:**
رغم أن GitHub يحفظ نسخاً من ملفاتك، من الجيد الاحتفاظ بنسخة احتياطية محلية من جميع ملفات الموقع.


## التخصيصات المتقدمة والميزات الإضافية

### ربط نطاق مخصص

إذا كنت تملك نطاقاً خاصاً بك (مثل www.myazhkar.com)، يمكنك ربطه بموقعك على GitHub Pages بدلاً من استخدام الرابط الافتراضي. هذا يعطي موقعك مظهراً أكثر احترافية ويجعله أسهل للتذكر.

**خطوات ربط النطاق المخصص:**

1. **شراء النطاق:** احصل على نطاق من مزود خدمة مثل GoDaddy أو Namecheap أو أي مزود آخر
2. **تكوين DNS:** في لوحة تحكم النطاق، أضف سجل CNAME يشير إلى `yourusername.github.io`
3. **تكوين GitHub:** في إعدادات GitHub Pages، أضف النطاق المخصص في حقل "Custom domain"
4. **تفعيل HTTPS:** تأكد من تفعيل خيار "Enforce HTTPS" لضمان الأمان

**مثال على تكوين DNS:**
```
Type: CNAME
Name: www
Value: yourusername.github.io
```

### إضافة Google Analytics

لمراقبة زوار موقعك وفهم كيفية استخدامهم له، يمكنك إضافة Google Analytics:

1. **إنشاء حساب Google Analytics:** اذهب إلى analytics.google.com وأنشئ حساباً جديداً
2. **إنشاء خاصية جديدة:** أضف موقعك كخاصية جديدة واحصل على معرف التتبع
3. **إضافة كود التتبع:** أضف كود Google Analytics إلى جميع صفحات HTML في قسم `<head>`

**مثال على كود Google Analytics:**
```html
<!-- Google Analytics -->

<script>

  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'GA_MEASUREMENT_ID');
</script>
```

### إضافة وضع عدم الاتصال

يمكنك إضافة Service Worker لجعل الموقع يعمل حتى بدون اتصال بالإنترنت:

```javascript
// service-worker.js
const CACHE_NAME = 'adhkar-v1';
const urlsToCache = [
    '/',
    '/morning.html',
    '/evening.html',
    '/morning-style.css',
    '/evening-style.css',
    '/morning-script.js',
    '/evening-script.js'
];

self.addEventListener('install', event => {
    event.waitUntil(
        caches.open(CACHE_NAME)
            .then(cache => cache.addAll(urlsToCache))
    );
});

self.addEventListener('fetch', event => {
    event.respondWith(
        caches.match(event.request)
            .then(response => response || fetch(event.request))
    );
});
```

### إضافة ميزة الوضع المظلم/الفاتح

يمكنك إضافة زر للتبديل بين الوضع المظلم والفاتح في جميع الصفحات:

```javascript
function toggleTheme() {
    const body = document.body;
    const currentTheme = body.getAttribute('data-theme');
    const newTheme = currentTheme === 'dark' ? 'light' : 'dark';
    
    body.setAttribute('data-theme', newTheme);
    localStorage.setItem('theme', newTheme);
}

// تحميل الثيم المحفوظ
document.addEventListener('DOMContentLoaded', () => {
    const savedTheme = localStorage.getItem('theme') || 'light';
    document.body.setAttribute('data-theme', savedTheme);
});
```


## صيانة الموقع والتحديثات المستقبلية

### إدارة التحديثات

بعد نشر موقعك، ستحتاج إلى تحديثه بانتظام للحفاظ على جودته وإضافة ميزات جديدة. GitHub يجعل هذه العملية سهلة ومباشرة.

**تحديث المحتوى:**
عندما تريد تحديث محتوى الأذكار أو إضافة أذكار جديدة، ما عليك سوى تحرير الملفات المناسبة مباشرة من واجهة GitHub. انقر على الملف الذي تريد تحديثه، ثم انقر على أيقونة القلم للتحرير. بعد إجراء التغييرات، اكتب رسالة commit وصفية واحفظ التغييرات.

**تحديث التصميم:**
إذا كنت تريد تغيير ألوان الموقع أو تحسين التصميم، يمكنك تحرير ملفات CSS مباشرة. تذكر أن أي تغيير تقوم به سيظهر على الموقع المباشر خلال بضع دقائق من حفظ التغييرات.

**إضافة صفحات جديدة:**
لإضافة صفحات جديدة (مثل صفحة للأدعية أو التسابيح)، أنشئ ملفات HTML جديدة واربطها بالصفحات الموجودة من خلال إضافة روابط في قوائم التنقل.

### نظام إدارة الإصدارات

GitHub يوفر نظام إدارة إصدارات قوي يساعدك في تتبع جميع التغييرات:

**استخدام Commits:**
كل تغيير تقوم به يُسجل كـ "commit" مع رسالة توضح ما تم تغييره. استخدم رسائل واضحة ووصفية مثل:
- "إضافة ذكر جديد لصفحة الصباح"
- "تحسين ألوان الثيم المسائي"
- "إصلاح مشكلة في عرض النصوص على الهواتف"

**استخدام Branches:**
للتغييرات الكبيرة، يمكنك إنشاء فرع جديد للعمل عليه دون التأثير على الموقع المباشر. بعد اختبار التغييرات، يمكنك دمجها مع الفرع الرئيسي.

**استخدام Releases:**
لإصدارات مهمة من موقعك، يمكنك إنشاء "release" يحتوي على ملخص للتغييرات الجديدة ورقم إصدار.

### النسخ الاحتياطية والأمان

**النسخ الاحتياطية التلقائية:**
GitHub يحتفظ بنسخ احتياطية من جميع ملفاتك تلقائياً. يمكنك الوصول إلى أي إصدار سابق من موقعك من خلال تاريخ الـ commits.

**النسخ الاحتياطية المحلية:**
رغم أن GitHub موثوق، من الجيد الاحتفاظ بنسخة محلية من ملفات موقعك على جهازك. يمكنك تحميل المستودع كملف ZIP من صفحة المستودع الرئيسية.

**أمان الحساب:**
- فعّل المصادقة الثنائية (2FA) على حساب GitHub الخاص بك
- استخدم كلمة مرور قوية وفريدة
- راجع صلاحيات الوصول بانتظام

### مراقبة الأداء

**مراقبة سرعة الموقع:**
استخدم أدوات مثل Google PageSpeed Insights لمراقبة سرعة موقعك بانتظام. إذا لاحظت تراجعاً في الأداء، تحقق من:
- حجم الصور المضافة حديثاً
- تعقيد ملفات CSS و JavaScript
- عدد الطلبات الخارجية (مثل الخطوط من Google Fonts)

**مراقبة الأخطاء:**
تحقق بانتظام من صفحة Actions في مستودعك للتأكد من عدم وجود أخطاء في عملية النشر. إذا ظهرت أخطاء، اقرأ تفاصيلها وقم بإصلاحها فوراً.

### التفاعل مع المستخدمين

**استقبال التعليقات:**
فعّل Issues في مستودعك للسماح للمستخدمين بإرسال اقتراحات أو الإبلاغ عن مشاكل. يمكنك الوصول إلى Issues من علامة تبويب "Issues" في مستودعك.

**إدارة المساهمات:**
إذا كنت تريد السماح للآخرين بالمساهمة في تحسين الموقع، يمكنك قبول Pull Requests من المطورين الآخرين. تأكد من مراجعة أي تغييرات مقترحة قبل قبولها.

### خطة الصيانة الدورية

**أسبوعياً:**
- تحقق من عمل الموقع على أجهزة مختلفة
- راجع أي تعليقات أو اقتراحات من المستخدمين
- تأكد من عدم وجود روابط معطلة

**شهرياً:**
- راجع إحصائيات الموقع إذا كنت تستخدم Google Analytics
- تحقق من تحديثات الأمان لأي مكتبات خارجية تستخدمها
- قم بنسخ احتياطي محلي للملفات

**سنوياً:**
- راجع محتوى الأذكار وتأكد من صحته
- فكر في إضافة ميزات جديدة بناءً على تعليقات المستخدمين
- قم بتحديث معلومات الاتصال والحقوق الفكرية

### التوسعات المستقبلية

**إضافة لغات أخرى:**
يمكنك إضافة ترجمات للموقع بلغات أخرى لتوسيع نطاق الجمهور المستهدف.

**تطبيق الهاتف المحمول:**
يمكنك تطوير تطبيق للهواتف الذكية يستخدم نفس المحتوى والتصميم.

**إضافة ميزات تفاعلية:**
مثل عداد للأذكار، تذكيرات يومية، أو إمكانية حفظ الأذكار المفضلة.

**التكامل مع وسائل التواصل:**
إضافة ميزات مشاركة متقدمة أو بوت للتليجرام أو واتساب.

هذا الدليل يوفر لك أساساً قوياً لنشر وإدارة موقع أذكار الصباح والمساء على GitHub Pages. تذكر أن النجاح في إدارة الموقع يأتي من الصيانة المنتظمة والاستجابة لاحتياجات المستخدمين. بارك الله فيك وتقبل منك هذا العمل الصالح.
